# 社区日报采集与索引维护 (Ingestion)

## 核心定位
本技能是社区知识库的"每日采集器"。
- **输入**: 飞书群聊原始记录（JSON 格式）
- **输出**: 6 大板块结构化日报 + 飞书播报 + 原始记录索引维护

## 📦 核心资产
1. **配置**: Cron 定时触发（每天固定时间运行）
2. **原始记录群**: 社区聊天记录存储群
3. **播报群**: hermes 同步播报群（需 @）
4. **知识库 Wiki Space ID**: `7629199638691597254`（名称：Hermes Agent 中文社区）

## 📋 日报格式规范

### 6 大板块（标题不可合并、不可改名、不可遗漏）
1. **📰 今日动态** — 当日重要事件、发布、更新
2. **📚 干货分享** — 技术文章、教程、资源分享
3. **🐛 踩坑实录** — 社区成员遇到的坑及解决方案
4. **🔥 热榜话题** — 群内讨论热度 TOP 6 话题（带引用）
5. **💬 群友原声** — 经典发言、金句、有价值讨论
6. **🏆 活跃 TOP5** — 当日最活跃成员

### 格式要求
- 顶部必须加**数据源声明**（"本日报基于飞书群聊记录自动生成，由 Hermes Agent 采集整理"）
- 6 大板块标题**完全一致，不可合并改名遗漏**
- 生成后必须用 `grep` 验证板块数量（应为 6 个），不匹配就打回重写

## 🛠️ 执行工作流

### 第一步：数据采集 (Fetch)
- 从原始记录群拉取当日飞书消息记录（JSON 格式）
- 多日期数据时，并行分派子代理处理

### 第二步：分析提炼 (Analyze)
- 分析 raw JSON，识别：
  - 重要事件/动态
  - 技术分享链接
  - 问题讨论与解决
  - 高频话题
  - 活跃用户
- 按 6 大板块分类整理

### 第三步：生成日报 (Generate)
- 使用 `doc_daily` 工具生成格式化日报文档
- 写入知识库 Wiki Space
- 维护原始记录索引表

### 第四步：播报 (Broadcast)
- 使用 `send-feishu-message.py` 将日报播报到播报群
- 播报需 @ 播报群
- 无事不扰原则：正常时极简播报，异常时详细汇报

## ⚠️ 实战注意事项
1. **子代理防偷懒**: 子代理生成格式化文档时会自动合并板块、改标题（如把 6 大板块改成"核心话题 TOP 3"）。必须在 prompt 中强调"标题完全一致、不可合并改名遗漏"。
2. **多日期并行**: 多日期数据必须使用 `delegate_task` 的 `tasks` 数组并行处理（最多 3 个同时），严禁串行。
3. **链接可点击**: 飞书消息中的链接需使用 `<a href="...">` 格式或确保 URL 自动识别，避免使用飞书不支持的 Markdown 链接语法。
